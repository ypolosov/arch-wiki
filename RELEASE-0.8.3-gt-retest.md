# arch-wiki v0.8.3 — ре-тест на `gt` (зеркало: чинит B-регрессию ссылок + порчу прозы)

> Поверх v0.8.2 (A/B/D-очистка git source-of-truth). Закрывает находки ре-теста 0.8.2:
> §1 был PARTIAL (B мисплейснут — 11 битых линков, tier-i не закрыт + рвал прозу).
> Схема не меняется (v2, миграции нет). Полный флоу — [RELEASE-0.8.0-gt-test.md](RELEASE-0.8.0-gt-test.md).

## Что изменилось (по находкам ре-теста 0.8.2)
- **Defect 1 (блокер tier-i) — порядок пайплайна.** `neutralizeRepoPaths` (B) теперь бежит **после**
  нейтрализатора repo-relative-ссылок и **до** C4-stub. Ссылка `[c4/src/model.c4](../c4/src/model.c4)`
  сперва сворачивается в inline-code-подпись `` `c4/src/model.c4` ``, и B выносит её начисто → `- Model:`
  без пути и **без битого пустого линка** `[](…)`. Закрывает все 11 линков (arc42 §3/§5/§6/§7,
  concepts/documentation-as-code, index.md). Подстраховка: пустую подпись `[](repo-url)` Core теперь
  дропает целиком.
- **Defect 2 (порча прозы).** (а) `(from ` `` `raw/x.md` `` `)` — провенанс-скобка теперь **backtick-aware**
  → удаляется целиком, без висячего `(from):`. (б) Связка-предлог, вводящая путь (`tracked in `` `risks.md` ``,
  `from raw/x.md`), удаляется **вместе с предлогом** → нет висячего `… in.` / `… readiness in.`.
  Многострочное не схлопывается (только горизонтальный пробел).
- **Minor — `**Source:**` с не-git остатком.** Вырезается только repo-путь из значения; Jira-ref
  (`GRM-3705`) и атрибуция эксперта **сохраняются** (CON-008/009/010). Строка дропается целиком только
  если после выреза не осталось ничего значимого.
- **Minor — голые корни в таблице слоёв index.md.** `c4/`, `raw/`, `.foam/`, `docs/architecture/` без
  хвоста пути теперь матчатся и вычищаются. Левый word-boundary не даёт зацепить `draw/`, `scc4/` и т.п.
  Косметика: голый корень, бывший единственным содержимым ячейки/буллета, оставляет её **пустой** (валидно).
